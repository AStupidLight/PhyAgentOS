# Robot Embodiment Declaration

This file describes the physical capabilities and constraints of the connected robot.
The Critic Agent reads this file to validate whether proposed actions are safe and feasible.

## Identity

- **Name**: Unitree G1
- **Type**: Humanoid mobile robot

## Sensors

- **Locomotion state**: Internal Unitree locomotion state machine
- **Odometry**: Not currently wired into this HAL driver
- **Vision**: Not currently wired into this HAL driver

## Supported Actions

| Action | Parameters | Description |
|--------|-----------|-------------|
| `primitive_motion` | `robot_id, primitive, distance_m?, angle_deg?, duration_s?, speed_scale?` | Execute a simple locomotion primitive such as `forward`, `backward`, `turn_left`, `turn_right`, or `turn_around` |
| `stop` | `robot_id` | Stop the current locomotion command |
| `connect_robot` | `robot_id` | Establish the control connection to the robot |
| `disconnect_robot` | `robot_id` | Close the current control connection |
| `reconnect_robot` | `robot_id` | Reset and restore the control connection |
| `check_connection` | `robot_id` | Run a lightweight heartbeat and update runtime connection state |
| `damp` | `robot_id` | Enter damp mode before posture transitions |
| `squat_to_stand` | `robot_id` | Stand up from the squat posture |
| `stand_to_squat` | `robot_id` | Move from standing posture to squat |
| `lie_to_stand` | `robot_id` | Stand up from lying posture |
| `low_stand` | `robot_id` | Switch to low stand height |
| `high_stand` | `robot_id` | Switch to high stand height |
| `zero_torque` | `robot_id` | Release torque output |
| `wave_hand` | `robot_id, turn_around?` | Trigger the built-in waving motion |
| `shake_hand` | `robot_id, stage?` | Trigger the built-in handshake motion |

## Connection

- **Transport**: DDS / Unitree SDK 2 Python
- **Network Interface**: configured by `network_interface`
- **Reconnect Policy**: manual by default

## Motion Capabilities

- **Recommended linear speed for primitives**: 0.25 m/s
- **Recommended angular speed for primitives**: 0.8 rad/s
- **Current implementation**: high-level `LocoClient` commands with approximate local pose integration
- **Not implemented here**: full perception, odometry feedback, manipulation, autonomous navigation

## Runtime Protocol

- **Connection channel**: `robots.g1_001.connection_state`
- **Pose channel**: `robots.g1_001.robot_pose`
- **Motion channel**: `robots.g1_001.motion_state`
- **Health owner**: `hal_watchdog.py` triggers connect on startup and periodic `health_check()`.

## Physical Constraints

- **Use case**: open-area supervised real-hardware experiments
- **Safety note**: do not issue locomotion or posture commands unless the robot has clear free space and a human spotter is present
